# Collaboration

Webifier is useful when it disappears into the workflow: add content, organize
it, push to Git, and let the website update itself. Contributions that make that
experience smoother are especially welcome.

If you try Webifier and something feels awkward, confusing, too manual, or just
not quite right, please say so. Feedback from real repositories is one of the
best ways to shape the project.

## Good Places To Help

- Improve templates for documentation, notebooks, galleries, and reports
- Add examples that show real project layouts
- Tighten the action and local CLI experience
- Improve notebook rendering and asset handling
- Write guides for common publishing patterns
- Test Webifier on existing repositories and report rough edges

## Design Philosophy

Webifier should stay small at the center and flexible at the edges. The core
should understand structure, loading, rendering, and output. Site-specific
choices such as layout, labels, navigation, comments, analytics, and content
types should remain easy to override.

## People

Webifier was created by Vahid Zehtab and Ahmad Salimi. The project is open to
issues, pull requests, documentation examples, and ideas from anyone trying to
make project publishing feel more automatic.

Feel free to reach out, open an issue, start a discussion, or send a pull
request. Questions, rough notes, examples of what did or did not work, and
early feedback are all welcome.

Start with the [build repository](https://github.com/webifier/build) or the
[website repository](https://github.com/webifier/webifier.github.io).
